prop 是单向绑定的：当父组件的属性变化时，将传导给子组件，但是不会反过来。这是为了防止子组件无意修改了父组件的状态——这会让应用的数据流难以理解。 另外，每次父组件更新时，子组件的所有 prop 都会更新为最新值。这意味着你不应该在子组件内部改变 prop 。如果你这么做了，Vue 会在控制台给出警告。 通常有两种改变 prop 的情况：


prop 作为初始值传入，子组件之后只是将它的初始值作为本地数据的初始值使用:

    props: ['initialCounter'],
    data: function () {
      return { counter: this.initialCounter }
    }

定义一个 computed 属性，此属性从 prop 的值计算得出。

    props: ['size'],
    computed: {
      normalizedSize: function () {
        return this.size.trim().toLowerCase()
      }
    }
